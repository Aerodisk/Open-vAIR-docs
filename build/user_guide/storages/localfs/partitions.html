  • Создать партицию на локальном диске
  • Просмотреть исходный код страницы

Данный функционал предоставляет возможность разбивать локальный несистемный диск на логические томы, удалять их и получать информацию о существующих партициях конкретного диска.

Создать партицию на системном диске нельзя. Так как данная процедура не гарантирует целостность системы после выполнения процедуры и перезагрузки.

Обратите внимание: При отсутствии таблицы партиций на диске, при первом обращении к списку партиций диска или при создании партиции, система автоматически создаст таблицу партиционирования формата gpt

Создать партицию на локальном диске

Чтобы создать партицию на локальном диске нужно воспользоваться эндпоинтом

POST: /storages/local-disks/create_partition/

{
«local_disk_path»: «/dev/sdb», #путь к жесткому диску
«storage_type»: «local_partition», #тип партиции
«start»: «0», #стартовое значение партиции на диске
«start_unit»: «GB», #единица измерения для стартового значения
«end»: «10», #конечное значение партиции на диске
«end_unit»: «GB» #единица измерения для конечного значения
}

Результатом успешного создания партиции будет такой json | { | «path»: «string», #путь к партиции | «size»: 0, #общий размер партиции | «mountpoint»: «string», #тчока монтирования партциии | «fs_uuid»: «string», #уникальный идентификатор | «type»: «string», #тип хранилища | «fstype»: «string», #файловая система хранилища | «parent»: «string» #диск на котором создана партиция | }

Получение информации о партициях на диске

Для получения информации о партцииях на локальном диске нужно воспользоваться эндпоинтом

GET: /storages/local-disks/partition_info/

данный эндпоинт в качестве входного значения принимает disk_path - путь до локального диска. Например /dev/sdb

результатом успешного выполнения данного эндпоинта будет словарь со всеми областями на диске, включая свободное пространство Обратите внимание, что нумерация в рамках словаря, не отражает номер партиции.

{
«0»: {
«Number»: «», #Номер партиции(отсутствует если это свободное место, см File system)
«Start»: «1024B», #стартовое значение партиции на диске
«End»: «500kB», #конечное значение партиции на диске
«Size»: «499kB», #общий размер партиции
«Type»: «», #тип партиции(отсутствует, если File system - Free Space)
«File system»: «Free Space», #файловая система партиции
«Flags»: «» #специальные метки отмечающие особенность партиции
},
«1»: {
«Number»: «1»,
«Start»: «500kB»,
«End»: «12.0GB»,
«Size»: «12,0GB»,
«Type»: «primary»,
«File system»: «»,
«Flags»: «»
},
«2»: {
«Number»: «»,
«Start»: «12.0GB»,
«End»: «21.5GB»,
«Size»: «9475MB»,
«Type»: «»,
«File system»: «Free Space»,
«Flags»: «»
}
}

Информация о возможных метах в колонке flags: https://www.gnu.org/software/parted/manual/html_node/set.html

Удаление партиции

ВНИМАНИЕ перед выполнением процедуры удаления партции, убедитесь, что вы сохранили важную информацию в другой раздел жёсткого диска

Для удаления партиции нужно воспользоваться энтрипоинтом

DELETE: /storages/local-disks/delete_partition/

Данный энтрипоинт принимает на вход json такого формата:

{
«storage_type»: «local_partition», #тип партиции
«local_disk_path»: «/dev/sdb», #путь до локального жесткого диска
«partition_number»: «string» #номер партиции
}

Результатом выполнения является сообщение об успешном удалении партиции
